## ВВЕДЕНИЕ И КОНТЕКСТ

### Бизнес-контекст и решаемая проблема

Компания предоставляет государственным органам информационную систему [eMedosmotr.kz](https://emedosmotr.kz/) для медицинского освидетельствования призывников военно-врачебной комиссией (ВВК) согласно [приказу](https://adilet.zan.kz/rus/docs/V2000021863).

**Как устроено внедрение:**
1. Заключается договор с Управлением мобилизационной подготовки области/района.
2. Инженеры-внедренцы выезжают в командировки: в **военкоматах** каждому врачу-члену ВВК устанавливают компьютер с доступом к ИС и проводят обучение.
3. В ближайшей **больнице (ЦРБ)** специалистам по инструментальным исследованиям настраивают доступ к личному кабинету, подключают медицинское оборудование к системе для автоматической загрузки результатов.
4. В регионах работают **региональные инженеры** — оказывают техподдержку во время призывной кампании.

**Две ключевые проблемы:**

| Проблема | Частота | Суть |
|----------|---------|------|
| Врачи не работают в системе | Редко | Не мотивированы делать заключения в ИС |
| Призывники уходят в частные клиники | Часто | Направление от врача ВВК есть, но анализы сдаются в платных клиниках — результаты не попадают в систему |

Для решения второй проблемы руководство пишет письма в гос. органы, убеждает членов ВВК направлять призывников в ЦРБ. **Нужен инструмент для отслеживания динамики по каждому объекту.**

**Особенности оборудования:**
- **Автоматическое подключение:** часть медоборудования подключается через перевалочный сервер (ноутбук + хаб рядом с оборудованием) — данные передаются автоматически.
- **Ручная загрузка:** часть оборудования не удаётся подключить — врачи, лаборанты, операторы сами загружают результаты в систему.

**Система eMedosmotr-map** — внутренний инструмент компании. Интерактивный геоинформационный дашборд для руководства и специалистов, обеспечивающий прозрачный контроль процессов внедрения и использования eMedosmotr по всему Казахстану.

## ЧАСТЬ 3. БАЗА ДАННЫХ (PostgreSQL + PostGIS)

### 3.1 Стратегия с геоданными — однократный перенос

Геоданные переносятся **один раз** из [`/Users/nurlan/Projects/maps_test_service`](/Users/nurlan/Projects/maps_test_service) в новую БД. После переноса проект `maps_test_service` можно удалить — никаких постоянных зависимостей не остаётся.

**Шаги переноса (выполняются при инициализации проекта):**

```bash
# 1. Дамп таблиц геоданных из maps_service
pg_dump -U nurlan -d maps_service \
  -t regions -t settlements \
  --data-only --column-inserts \
  -f scripts/geo_seed.sql

# 2. Импорт в новую БД (после создания таблиц через Alembic)
psql -U postgres -d emedosmotr_map -f scripts/geo_seed.sql
```

Файл `geo_seed.sql` коммитится в репозиторий нового проекта как начальный seed — это удобно для воспроизводимости dev-окружения и CI.

```python
# config.py — один движок, без зависимости от maps_service
MAIN_DATABASE_URL = "postgresql+asyncpg://postgres:password@localhost:5432/emedosmotr_map"
```

### 3.2 Схема основной БД `emedosmotr_map`

```sql
-- ПОЛЬЗОВАТЕЛИ И РОЛИ
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    full_name VARCHAR(200),
    role VARCHAR(30) NOT NULL CHECK (role IN (
        'superadmin','director','regional_manager','engineer','operator','analyst'
    )),
    region_id INTEGER,                 -- Привязка к региону (для regional_manager, engineer)
    is_active BOOLEAN DEFAULT true,
    avatar_url TEXT,
    phone VARCHAR(20),
    last_login_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT now(),
    updated_at TIMESTAMPTZ DEFAULT now(),
    deleted_at TIMESTAMPTZ
);

-- ОБЪЕКТЫ НА КАРТЕ (ЦРБ, военкоматы, клиники)
-- Добавляются вручную сотрудниками, с указанием наименования и координат
CREATE TABLE locations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    settlement_id INTEGER,             -- FK -> maps_service.settlements (необязательно)
    region_id INTEGER NOT NULL,        -- FK -> maps_service.regions
    name VARCHAR(300) NOT NULL,
    type VARCHAR(50) NOT NULL CHECK (type IN (
        'military_office',             -- Военкомат (ВВК)
        'district_hospital',           -- ЦРБ / больница (место загрузки результатов)
        'private_clinic',              -- Частная клиника (куда уходят призывники)
        'medical_center',              -- Медцентр
        'relay_server_location'        -- Точка с перевалочным сервером (ноутбук + хаб)
    )),
    address TEXT,
    lat NUMERIC(10,7),                 -- Координаты указываются вручную при добавлении
    lon NUMERIC(10,7),
    geom GEOMETRY(Point, 4326),        -- PostGIS для пространственных запросов
    -- СТАТУС АНАЛИЗОВ (ключевая метрика):
    -- ready       = анализы прикрепляются исправно (зелёный)
    -- in_progress = часть работает, часть нет (жёлтый)
    -- critical    = призывники уходят в сторонние клиники / нет данных (красный)
    status VARCHAR(20) NOT NULL DEFAULT 'critical' CHECK (status IN (
        'ready', 'in_progress', 'critical'
    )),
    status_reason TEXT,
    -- ТИП ЗАГРУЗКИ ДАННЫХ:
    upload_mode VARCHAR(30) DEFAULT 'manual' CHECK (upload_mode IN (
        'auto',                        -- Перевалочный сервер (автоматически)
        'manual',                      -- Врачи/лаборанты загружают вручную
        'mixed'                        -- Часть автоматически, часть вручную
    )),
    has_relay_server BOOLEAN DEFAULT false,  -- Есть ли перевалочный сервер
    relay_server_notes TEXT,           -- Модель ноутбука, IP, примечания
    is_active BOOLEAN DEFAULT true,
    notes TEXT,                        -- Произвольные примечания сотрудников
    meta JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT now(),
    updated_at TIMESTAMPTZ DEFAULT now(),
    deleted_at TIMESTAMPTZ
);
CREATE INDEX idx_locations_geom ON locations USING GIST(geom);
CREATE INDEX idx_locations_region ON locations(region_id);
CREATE INDEX idx_locations_status ON locations(status);

-- ПРИЗЫВНЫЕ КОМИССИИ (привязаны к объекту)
CREATE TABLE commissions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    location_id UUID NOT NULL REFERENCES locations(id) ON DELETE CASCADE,
    address TEXT,
    computers_available INTEGER DEFAULT 0,
    computers_required INTEGER DEFAULT 0,
    internet_status BOOLEAN DEFAULT false,
    internet_speed_mbps NUMERIC(8,2),
    status VARCHAR(20) NOT NULL DEFAULT 'critical',
    comment TEXT,
    last_updated_by UUID REFERENCES users(id),
    last_updated_at TIMESTAMPTZ DEFAULT now(),
    created_at TIMESTAMPTZ DEFAULT now()
);

-- МЕДИЦИНСКИЕ ОРГАНИЗАЦИИ
CREATE TABLE medical_organizations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    location_id UUID NOT NULL REFERENCES locations(id) ON DELETE CASCADE,
    name VARCHAR(300) NOT NULL,
    address TEXT,
    internet_status BOOLEAN DEFAULT false,
    status VARCHAR(20) NOT NULL DEFAULT 'critical',
    comment TEXT,
    last_updated_by UUID REFERENCES users(id),
    last_updated_at TIMESTAMPTZ DEFAULT now(),
    created_at TIMESTAMPTZ DEFAULT now()
);

-- МЕДИЦИНСКОЕ ОБОРУДОВАНИЕ
CREATE TABLE medical_equipment (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    organization_id UUID NOT NULL REFERENCES medical_organizations(id) ON DELETE CASCADE,
    equipment_type VARCHAR(30) NOT NULL CHECK (equipment_type IN (
        'ultrasound','xray','ct','mri','ecg','other'
    )),
    count INTEGER DEFAULT 0,
    free_ports INTEGER DEFAULT 0,
    connection_status BOOLEAN DEFAULT false,
    port_status VARCHAR(20) DEFAULT 'free' CHECK (port_status IN ('free','occupied')),
    ecg_model VARCHAR(100),
    smart_ecg_compatible BOOLEAN DEFAULT false,
    serial_number VARCHAR(100),
    last_service_date DATE,
    notes TEXT,
    updated_at TIMESTAMPTZ DEFAULT now()
);

-- АНАЛИТИКА: ВОРОНКА НАПРАВЛЕНИЙ
-- Ключевая метрика: направлений выдано ВВК → результатов загружено в систему
-- Потери = призывники ушли в частные клиники (данные не попали в систему)
CREATE TABLE funnel_data (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    location_id UUID NOT NULL REFERENCES locations(id),
    region_id INTEGER NOT NULL,
    period_year INTEGER NOT NULL,
    period_month INTEGER NOT NULL,
    -- Основные показатели
    directions_issued INTEGER DEFAULT 0,        -- Выдано направлений от ВВК врачей
    results_uploaded INTEGER DEFAULT 0,         -- Загружено результатов в eMedosmotr
    results_via_auto INTEGER DEFAULT 0,         -- Из них: через перевалочный сервер (авто)
    results_via_manual INTEGER DEFAULT 0,       -- Из них: загружено вручную врачами
    results_via_external INTEGER DEFAULT 0,     -- Ушли в сторонние клиники (потери)
    doctors_active INTEGER DEFAULT 0,           -- Врачей-членов ВВК, работавших в системе
    doctors_total INTEGER DEFAULT 0,            -- Общее количество врачей ВВК
    -- Расчётные показатели (заполняются триггером)
    conversion_rate NUMERIC(5,2),               -- results_uploaded / directions_issued * 100
    loss_rate NUMERIC(5,2),                     -- results_via_external / directions_issued * 100
    -- Источник данных
    source VARCHAR(50) DEFAULT 'manual' CHECK (source IN (
        'etl_emedosmotr',                       -- Автосинхронизация из eMedosmotr
        'manual'                                -- Внесено вручную сотрудником
    )),
    synced_at TIMESTAMPTZ,
    notes TEXT,                                 -- Причины потерь, комментарии
    created_at TIMESTAMPTZ DEFAULT now(),
    UNIQUE(location_id, period_year, period_month)
);

-- ЗАДАЧИ (Tasks)
CREATE TABLE tasks (
```

### 4.3 WebSocket менеджер

```python
# modules/ws/manager.py
class ConnectionManager:
    def __init__(self):
        self.active_connections: dict[str, list[WebSocket]] = {}
        self.room_connections: dict[str, set[WebSocket]] = {}

    async def connect(self, ws: WebSocket, user_id: str, rooms: list[str]):
        await ws.accept()
        for room in rooms:
            self.room_connections.setdefault(room, set()).add(ws)

    async def broadcast_to_room(self, room: str, event: str, data: dict):
        message = {"event": event, "data": data}
        for ws in self.room_connections.get(room, set()).copy():
            try:
                await ws.send_json(message)
            except WebSocketDisconnect:
                self.room_connections[room].discard(ws)

# Комнаты: "region_{id}", "role_{role_name}", "user_{user_id}"
```

**Авторизация WebSocket:** JWT передаётся в query-параметре `?token=...` при подключении. Сервер валидирует токен до `accept()`, при невалидном токене — `close(4001)`.

**Heartbeat:** сервер отправляет `ping` каждые 30 секунд. Если `pong` не получен за 10 секунд — соединение закрывается.

**Клиентский reconnect:**
```typescript

```

#### Web Push (VAPID)

**Backend:**

```python
# modules/notifications/push_service.py
from pywebpush import webpush, WebPushException

class PushService:
    VAPID_PRIVATE_KEY = settings.VAPID_PRIVATE_KEY
    VAPID_PUBLIC_KEY  = settings.VAPID_PUBLIC_KEY
    VAPID_CLAIMS = {"sub": "mailto:admin@emedosmotr.kz"}

    async def send(self, subscription: PushSubscription, title: str, body: str, url: str = "/"):
        payload = json.dumps({"title": title, "body": body, "url": url})
        try:
            webpush(
                subscription_info={"endpoint": subscription.endpoint,
                                   "keys": {"p256dh": subscription.p256dh, "auth": subscription.auth}},
                data=payload,
                vapid_private_key=self.VAPID_PRIVATE_KEY,
                vapid_claims=self.VAPID_CLAIMS,
            )
        except WebPushException as e:
            if e.response and e.response.status_code == 410:
                # Подписка устарела — деактивируем
                await subscription.deactivate()
            raise
```

#### Telegram-уведомления

**Схема привязки аккаунта:**

```
1. Пользователь нажимает «Привязать Telegram» в настройках профиля
2. Backend генерирует одноразовый код (UUID, TTL 10 минут) → хранит в Redis
3. Пользователь отправляет боту команду: /bind <код>
4. Бот получает код → проверяет в Redis → сохраняет chat_id в telegram_bindings
5. Подтверждение в чате бота + Toast в интерфейсе
```

```python
# modules/notifications/telegram_bot.py
import httpx

TELEGRAM_API = f"https://api.telegram.org/bot{settings.TELEGRAM_BOT_TOKEN}"

async def send_message(chat_id: int, text: str, parse_mode: str = "HTML") -> bool:
    async with httpx.AsyncClient() as client:
        resp = await client.post(f"{TELEGRAM_API}/sendMessage", json={
            "chat_id": chat_id,
            "text": text,
            "parse_mode": parse_mode,
        })
        return resp.status_code == 200

async def handle_webhook(update: dict):
    message = update.get("message", {})
    text = message.get("text", "")
    chat_id = message["chat"]["id"]
    username = message["from"].get("username")

    if text.startswith("/bind "):
        code = text.split(" ", 1)[1].strip()
        user_id = await redis.get(f"telegram_bind:{code}")
        if user_id:
            await save_telegram_binding(user_id, chat_id, username)
            await redis.delete(f"telegram_bind:{code}")
            await send_message(chat_id, "✅ Аккаунт успешно привязан к eMedosmotr Map!")
        else:
            await send_message(chat_id, "❌ Код недействителен или истёк. Запросите новый в профиле.")
```

### 4.8 Расчёт расстояний и логистика

#### Назначение

Инструмент позволяет быстро оценить **дорожное расстояние и время в пути** между любыми двумя (или более) объектами на карте — для планирования командировок инженеров, расстановки приоритетов выездов, построения оптимального маршрута по нескольким точкам.

#### Источник данных маршрутов

Используется **Yandex Maps Router API** — тот же ключ, что уже применяется для карты. Поддерживает Казахстан, возвращает реальное дорожное расстояние, время в пути и геометрию маршрута.

```
Режимы: driving (авто) | transit (общественный транспорт) | walking (пешком)
Ответ:  distance_km, duration_min, encoded polyline
```

Альтернатива для быстрой оценки (без API-запроса): PostGIS `ST_Distance` — прямолинейное (воздушное) расстояние, умноженное на эмпирический коэффициент дорожной извилистости `1.3`. Используется как fallback при недоступности Yandex API.

---

#### Backend — `routing` модуль

```python
# modules/routing/service.py
import httpx
from math import radians, sin, cos, sqrt, atan2

YANDEX_ROUTER_URL = "https://api.routing.yandex.net/v2/route"

class RoutingService:

    async def get_route(
        self,
        origin: tuple[float, float],    # (lat, lon)
        dest: tuple[float, float],
        mode: str = "driving",
    ) -> RouteResult:
        # 1. Проверить кэш (совпадение с точностью до 4 знаков ≈ 11м)
        cached = await self._get_cached(origin, dest, mode)
        if cached:
            return cached

        # 2. Запрос к Yandex Router API
        try:
            result = await self._yandex_route(origin, dest, mode)
        except Exception:
            # 3. Fallback — воздушное расстояние × 1.3
            result = self._haversine_estimate(origin, dest)

        # 4. Сохранить в кэш
        await self._save_cache(origin, dest, mode, result)
        return result

    def _haversine_estimate(self, origin, dest) -> RouteResult:
        """Прямолинейное расстояние (Haversine) × 1.3 коэффициент дороги."""
        R = 6371
        lat1, lon1 = map(radians, origin)
        lat2, lon2 = map(radians, dest)
        dlat, dlon = lat2 - lat1, lon2 - lon1
        a = sin(dlat/2)**2 + cos(lat1)*cos(lat2)*sin(dlon/2)**2
        straight_km = R * 2 * atan2(sqrt(a), sqrt(1-a))
        road_km = round(straight_km * 1.3, 1)
        duration_min = int(road_km / 60 * 60)  # ~60 км/ч
        return RouteResult(distance_km=road_km, duration_min=duration_min,
                           polyline=None, is_estimate=True)

    async def get_multi_stop_route(
        self, waypoints: list[tuple[float, float]], mode: str = "driving"
    ) -> MultiStopResult:
        """
        Маршрут через N точек с оптимизацией порядка посещения
        (nearest-neighbour эвристика для <= 10 точек).
        """
        if len(waypoints) <= 2:
            result = await self.get_route(waypoints[0], waypoints[-1], mode)
            return MultiStopResult(ordered_waypoints=waypoints, legs=[result],
                                   total_distance_km=result.distance_km,
                                   total_duration_min=result.duration_min)
        optimized = self._nearest_neighbour(waypoints)
        legs = []
        for i in range(len(optimized) - 1):
            legs.append(await self.get_route(optimized[i], optimized[i+1], mode))
        return MultiStopResult(
            ordered_waypoints=optimized,
            legs=legs,
            total_distance_km=sum(l.distance_km for l in legs),
            total_duration_min=sum(l.duration_min for l in legs),
        )

    async def get_distance_matrix(
        self, location_ids: list[UUID]
    ) -> dict[str, dict[str, RouteResult]]:
        """Матрица «всех до всех» — кэшируется агрессивно."""
        pass
```

### 10.6 Soft Delete

Все таблицы с `deleted_at` используют middleware-фильтр:

```python
# services/base.py
class BaseService:
    async def get_query(self):
        """Автоматически исключает мягко удалённые записи"""
        return select(self.model).where(self.model.deleted_at.is_(None))

    async def soft_delete(self, id: UUID, user_id: UUID):
        """Мягкое удаление + audit log"""
        await self.update(id, {"deleted_at": datetime.utcnow()})
        await self.audit_log("delete", id, user_id)
```
