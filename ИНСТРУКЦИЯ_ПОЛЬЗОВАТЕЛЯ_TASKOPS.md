# Руководство пользователя: Модуль операционного управления (TaskOps)

Добро пожаловать в модуль **TaskOps** системы eMedosmotr-map. Этот модуль предназначен для управления проектами, задачами и стратегическими целями команды, объединяя в себе лучшие практики таких инструментов, как Linear, Jira и Notion, с уникальной привязкой к объектам медицинской инфраструктуры.

---

## 🚀 Быстрый старт: С чего начать?

Если вы впервые открыли модуль и видите пустой экран «Мои задачи», следуйте этим шагам:

1.  **Создайте первый проект:**
    - В левой боковой панели в секции **«Проекты и поручения»** нажмите на иконку **«+»**.
    - Либо нажмите `⌘` + `K` (командная палитра) и выберите (или нажмите клавишу `P`) **«Создать проект»**.
    - Введите название проекта (например, «Развертывание системы») и нажмите Enter.

2.  **Выберите проект:**
    - После создания проект появится в списке слева. Нажмите на него, чтобы открыть рабочее пространство проекта.

3.  **Переключитесь на нужный инструмент (Gantt, Board и др.):**
    - В верхней части открытого проекта вы увидите вкладки: **Список**, **Доска**, **Таймлайн (Gantt)**, **Календарь**, **Спринты**.
    - Нажмите на **«Таймлайн»**, чтобы увидеть диаграмму Ганта.

4.  **Создайте задачи:**
    - Внутри проекта нажмите синюю кнопку **«+ Задача»** в правом верхнем углу.
    - Или используйте командную палитру: `⌘` + `K` -> клавиша `C`.
    - Вы также можете применить готовый **«📋 Шаблон»**, чтобы мгновенно создать структуру проекта.

## 1. Переключение пространств (Карта ↔ Задачи)

Модуль TaskOps реализован как отдельное рабочее пространство. Это позволяет полностью сфокусироваться на работе с задачами, отключая тяжелые картографические ресурсы.

- **Как переключиться:** В левой части верхней панели (Header) находится переключатель. Нажмите на **«Задачи»**, чтобы войти в модуль управления, или на **«Карта»**, чтобы вернуться к гео-мониторингу.
- **Горячая клавиша:** Используйте `⌘` + `.` (точка) для быстрого переключения между режимами.

> [!NOTE]
> При переходе в режим «Задачи» карта полностью выгружается из памяти браузера, что обеспечивает максимальную производительность интерфейса.

---

## 2. Личное пространство (Inbox)

**Inbox (Входящие)** — это центральный хаб для каждого исполнителя. Здесь собраны все задачи, требующие вашего внимания.

- **Секции:** Задачи сгруппированы по времени: «Сегодня», «На этой неделе», «В планах» и «Без даты».
- **Фокус:** В Inbox попадают только те задачи, где вы являетесь исполнителем или наблюдателем.
- **Быстрые действия:** Вы можете менять статус задачи прямо из списка или открыть панель деталей кликом по строке.

---

## 3. Разовые поручения (без проекта)

Если вам нужно быстро дать кому-то задание, не создавая полноценный проект, — используйте функцию **«Дать поручение»**.

### Как это работает

- В левой боковой панели, **под строкой поиска**, всегда видна кнопка **«📝 Дать поручение»** (пунктирная рамка).
- Нажмите на неё — откроется форма:
  1. **Суть поручения** — кратко опишите задачу (обязательное поле).
  2. **Подробности** — дополнительный контекст, ссылки, требования (необязательно).
  3. **👤 Исполнитель** — выберите сотрудника из выпадающего списка.
  4. **📅 Срок выполнения** — установите дедлайн.
  5. **Срочность** — выберите один из 4 уровней: Срочно / Высокий / Средний / Низкий.
- Нажмите **«📝 Дать поручение»** или `Enter`.

> [!NOTE]
> Поручения автоматически попадают в системный раздел **«📝 Поручения»**, который появится в боковой панели сразу после первого поручения. Это отдельный «лоток» для разовых задач, отделённый от полноценных проектов.

### После создания

- Откройте задачу и назначьте **исполнителя** (кнопка «Назначить» в боковой панели задачи).
- Исполнитель получит push-уведомление и увидит поручение в своём разделе **«Мои задачи»**.
- Вы можете отслеживать все поручения, кликнув на «📝 Поручения» в сайдбаре.

> [!TIP]
> Раздел «Поручения» поддерживает все те же инструменты, что и обычный проект: **Список**, **Доска**, **Таймлайн (Ганта)** и **Календарь**.

## 4. Управление проектами и целями

### 4.1 Стратегические цели (Goals)
Раздел **«Цели»** позволяет отслеживать глобальные инициативы компании.
- Каждая цель имеет индикатор прогресса (кольцо), который рассчитывается автоматически на основе завершенных проектов и задач.
- Цели могут объединять несколько проектов.

### 4.2 Проекты (Projects)
Проекты — это основные контейнеры для задач (например, «Внедрение в Акмолинской области» или «Разработка версии 2.0»).
- **Шаблоны проектов:** Чтобы не создавать типовые задачи вручную, используйте кнопку **«📋 Шаблон»** внутри проекта. Доступны шаблоны:
    - *Внедрение в области* (автоматическое создание 30+ этапов внедрения).
    - *Продуктовый спринт* (структура для разработчиков).
    - *Инцидент на объекте*.

---

## 4. Режимы отображения (Views)

Внутри любого проекта вы можете переключаться между пятью режимами отображения одних и тех же данных:

1.  **Список (List):** Классический вид с группировкой по статусам. Идеален для быстрой пробежки по бэклогу.
2.  **Доска (Board):** Канбан-доска. Перетаскивайте карточки между колонками (Drag-n-Drop) для смены статуса.
3.  **Таймлайн (Gantt):** Визуальный график Ганта.
    - Устанавливайте сроки простым растягиванием баров.
    - Связывайте задачи стрелками зависимостей (одна задача блокирует другую).
4.  **Календарь (Calendar):** Сетка месяца с распределением задач по датам.
5.  **Спринты (Cycles):** Режим для итерационной разработки. Позволяет планировать работу на 2 недели, отслеживать график сгорания задач (Burndown chart) и автоматически переносить незавершенные дела в следующий спринт.

## 5. Работа с задачей

### 5.1 Панель деталей
При клике на задачу справа открывается панель (Side-panel), где можно:
- **Редактирование «на лету»:** Просто кликните на заголовок или описание, чтобы изменить их. Сохранение происходит автоматически при выходе из поля (потере фокуса).
- **Установка параметров:** Выбирайте статус, приоритет и исполнителя из выпадающих списков.
- **Управление датами:** Устанавливайте «Срок» (Due date) и «Дату начала» через календарь.
- **Оценки:** Указывайте сложность в часах или T-shirt размерах (XS, S, M, L, XL).
- **Сброс значений:** Рядом с датами и исполнителем есть иконка **«×»**, позволяющая быстро очистить поле.

### 5.2 Привязка к карте
Это уникальная функция TaskOps. Задачу можно привязать к конкретному объекту (Военкомат, ЦРБ).
- В задаче появится ссылка на объект.
- При клике на ссылку система переключит вас в режим «Карта» и центрирует экран на выбранном объекте.
- На самой карте в карточке объекта во вкладке «Задачи» будет видна вся история связанных с ним работ.

### 5.3 Комментарии и @упоминания
- Обсуждайте задачи в блоке комментариев.
- Используйте **@имя_пользователя**, чтобы отправить коллеге мгновенное уведомление.
- К комментариям можно прикреплять фотографии и документы (особенно актуально для инженеров на выезде).

## 6. Аналитика и контроль

### 6.1 Дашборд руководителя
Экран аналитики предоставляет сводку по всей операционной деятельности:
- **KPI-карточки:** Общее кол-во открытых, просроченных и завершенных задач.
- **Throughput:** График производительности команды по неделям.
- **Risk-list:** Список задач, которые с высокой вероятностью будут просрочены.

#### 📋 Просмотр задач конкретного сотрудника

В блоке **«Нагрузка по исполнителям»** отображается список всех сотрудников с количеством их активных задач. Цвет счётчика показывает уровень загрузки:
- 🟢 **Зелёный** — до 4 задач (норма)
- 🟡 **Жёлтый** — 5–7 задач (высокая нагрузка)
- 🔴 **Красный** — 8+ задач (перегрузка)

**Как посмотреть задачи сотрудника:**
1. Нажмите на строку с именем исполнителя в блоке «Нагрузка».
2. Справа откроется панель со списком всех его **активных задач** — из всех проектов и поручений одновременно.
3. Нажмите на любую задачу, чтобы открыть панель деталей и отредактировать её.
4. Включите чекбокс **«Показать завершённые»**, чтобы увидеть полную историю.

> [!TIP]
> Просроченные задачи будут выделены красным со значком ⚠️.


### 6.2 Журнал аудита (Audit Log)
Для обеспечения безопасности и прозрачности (особенно при работе с внешними разработчиками) ведется полный лог действий:
- Кто, когда и какую правку внес.
- История смены статусов и назначений.

## 7. Горячие клавиши (Keyboard-first)

Для опытных пользователей предусмотрено управление без мыши:

- `⌘` + `K` — Командная палитра (быстрый поиск задач, проектов или создание новой задачи).
- `C` — Создать новую задачу.
- `S` — Сменить статус выбранной задачи.
- `1`, `2`, `3`, `4` — Установить приоритет P0–P3.
- `A` — Назначить исполнителя.

---

## 8. Мобильная версия

Модуль оптимизирован для работы «в полях»:
- **Интерфейс инженера:** Упрощенный список задач и Inbox.
- **Свайпы:** Быстрое закрытие задачи свайпом вправо.
- **Фото:** Быстрая загрузка фотографий выполненных работ прямо с камеры телефона.
- **Уведомления:** Push-уведомления и интеграция с Telegram-ботом.
